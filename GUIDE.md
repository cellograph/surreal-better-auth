## 1. Install

```bash

```

## 2. Connect to SurrealDB

```ts
// lib/db.ts
import Surreal from "surrealdb";

const db = new Surreal();

await db.connect("ws://localhost:8000");
await db.use({ namespace: "myapp", database: "production" });

export { db };
```

---

## 3. Configure Better Auth

```ts
// lib/auth.ts
import { betterAuth } from "better-auth";
import { surrealdbAdapter } from "surreal-better-auth";
import { db } from "./db";

export const auth = betterAuth({
  database: surrealdbAdapter(db, {
    idGenerator: "surreal.ULID", // recommended for v3
    usePlural: false,            // use 'user' not 'users'
  }),
  emailAndPassword: { enabled: true },
});
```

---

## 4. Adapter Options

| Option | Type | Default | Description |
|---|---|---|---|
| `idGenerator` | see below | `undefined` | ID generation strategy |
| `usePlural` | `boolean` | `false` | Use `users` instead of `user` |
| `allowPassingId` | `boolean` | `false` | Allow custom IDs in create calls |
| `debugLogs` | `boolean` | `false` | Log all SurrealQL queries |

### `idGenerator` values

| Value | Generated by | Result |
|---|---|---|
| `"surreal.ULID"` | Database | `ulid()` â€” sortable, recommended |
| `"surreal.UUIDv7"` | Database | `uuid v7` â€” time-based |
| `"surreal.UUIDv4"` | Database | `uuid v4` â€” random |
| `"surreal.UUID"` | Database | default uuid |
| `"surreal.guid"` | Database | 20-char alphanumeric |
| `"surreal"` | Database | SurrealDB default |
| `"sdk.UUIDv4"` | SDK | UUID v4 via JS SDK |
| `"sdk.UUIDv7"` | SDK | UUID v7 via JS SDK |
| `undefined` | Better Auth | Better Auth default |

### ID generation precedence

1. `advanced.database.generateId()` â€” highest priority, overrides everything
2. `idGenerator` â€” used if `generateId()` is not defined
3. Custom ID from data â€” used if `allowPassingId: true` and ID is provided
4. Better Auth default â€” used if `allowPassingId: true` and no ID in data
5. Database default â€” fallback (same as `idGenerator: "surreal"`)

## 5. Generate Schema

Use the Better Auth CLI to generate your SurrealDB schema:

```bash
bunx @better-auth/cli generate --output schema.surql
```

This outputs a `.surql` file ready to apply to your database. Apply it by pasting into **Surrealist** or running:

```bash
surreal import --conn ws://localhost:8000 --user root --pass root \
  --ns myapp --db production schema.surql
```

> **Note for v3:** The generated schema no longer includes `DEFINE FIELD id` or
> `DEFINE INDEX ... COLUMNS id` â€” both are invalid in SurrealDB v3. The adapter
> handles this automatically.


## 7. Important SurrealDB v3 Behaviour Changes

- **Non-existent tables throw errors** â€” always apply your schema before running the app
- **Hyphenated IDs are escaped** â€” `user:my-id` becomes `user:âź¨my-idâź©` in v3; prefer ULID/UUID generators to avoid this
- **`DELETE` on empty table** â€” fine; `DELETE` on undefined table â€” throws

## Publishing to npm

### One-time setup

```bash
# Create an account at https://npmjs.com, then:
npm login
```

### Checklist before publishing

**1. Bump the version** in `packages/surreal-better-auth/package.json`:

```json
{ "version": "1.1.0" }
```

Or use npm to bump automatically:

```bash
cd packages/surreal-better-auth
npm version patch   # 1.0.0 â†’ 1.0.1  (bug fix)
npm version minor   # 1.0.0 â†’ 1.1.0  (new feature)
npm version major   # 1.0.0 â†’ 2.0.0  (breaking change)
```

**2. Verify what will be published:**

```bash
cd packages/surreal-better-auth
npm pack --dry-run
```

Only `dist/`, `LICENSE`, `README.md`, and `package.json` should be included (controlled by the `"files"` field in `package.json`).

**3. Run tests one final time:**

```bash
bun run --filter=surreal-better-auth test
```

**4. Build:**

```bash
bun run build
```

**5. Publish:**

```bash
cd packages/surreal-better-auth
npm publish
```

For a public scoped package (e.g. `@yourname/surreal-better-auth`):

```bash
npm publish --access public
```
